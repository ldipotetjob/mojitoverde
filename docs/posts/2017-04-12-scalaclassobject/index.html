Scala - Classes - Objects

2017-04-12
6 minute read
scala programming
Scala

In this post we talk about Scala Class, Objects, Traits and Interfaces. Its relationships between different components and inheritance. We are assuming that classes, methods, and polymorphism are familiar to the reader and we will only delve into those terms that are new to the scenario at hand.

The next example or its idea have been taken from Programming in scala, 3rd edition

Defining classes in scala like in any other language is a blueprint for creating objects.

Let’s define a very simple implementation of the Rational numbers, just a few simple operations:

 1// Modeling Rational numbers 
 2class Rational(n: Int, d: Int) {
 3  // rise java.lang.IllegalArgumentException
 4  require(d != 0)
 5  private val g = gcd(n.abs, d.abs)
 6  val numer = n / g
 7  val denom = d / g
 8  def this(n: Int) = this(n, 1)
 9
10  // addition  operation between rational numbers
11  def + (that: Rational): Rational =
12    new Rational(
13      numer * that.denom + that.numer * denom,
14      denom * that.denom
15    )
16  // addition  operation between rational number and integer number
17  def + (i: Int): Rational =
18    new Rational(numer + i * denom, denom)
19
20  // subtraction operation between rational numbers
21  def - (that: Rational): Rational =
22    new Rational(
23      numer * that.denom - that.numer * denom,
24      denom * that.denom
25    )
26  // subtraction operation between rational number and integer number
27  def - (i: Int): Rational =
28    new Rational(numer - i * denom, denom)
29
30  // multiplication between rational numbers
31  def * (that: Rational): Rational =
32    new Rational(numer * that.numer, denom * that.denom)
33  // multiplication between rational num and int number
34  def * (i: Int): Rational =
35    new Rational(numer * i, denom)
36
37  //  division between rational numbers
38  def / (that: Rational): Rational =
39    new Rational(numer * that.denom, denom * that.numer)
40  //  division between rational number and integer number
41  def / (i: Int): Rational =
42    new Rational(numer, denom * i)
43
44  // we override here the default definition of toString method
45  override def toString = s"$numer/$denom"
46
47  // gcd: greatest common divisor
48  // because "private" it can only be call inside the class
49  private def gcd(a: Int, b: Int): Int =
50    if (b == 0) a else gcd(b, a % b)
51}
52
53val rational: Rational = new Rational(3,4) //rational: Rational = 3/4
54val rational1: Rational = new Rational(5,6) //rational1: Rational = 5/6
55// Normal scenario
56rational + rational1 //res0: Rational = 19/12
57rational - rational1 //res1: Rational = -1/12
58
59rational*(2) //res2: Rational = 3/2
60rational+(3) res3: Rational = 15/4
61

So basically a solution can be implicit convertion. Implicit definitions are those that the compiler is allowed to insert into a program in order to fix any of its type errors. For example, if x * Rational_number does not type check, then the compiler might change it to convert(x) * Rational_number, where convert is some available implicit conversion. If convert changes x into something that has a * method that accept Rational Number, then this change might fix a program so that it type checks and runs correctly.

Places where implicit convertions are implemented:

  1. Conversions to an expected type. It lets you use one type in a context where a different type is expected
  2. Conversions of the receiver of a selection. It lets you adapt the receiver of a method call (i.e., the object on which a method is invoked), if the method is not applicable on the original type
  3. Implicit parameters. They are usually used to provide more information to the called function about what the caller wants. Useful with generic functions.

An object is a class that has exactly one instance. It is created lazily when it is referenced, like a lazy val. In scala doesn’t exist static values but we have singleton objects which can be imported from everywhere in our code(objects). Every NOT private method can be used for any instant that imports the object previously.

1// Singleton object
2object Person {
3  import java.time.LocalDate
4  val year = LocalDate.now.getYear
5  private def calculateYearOfBirth(yearOfBirth:Int): Int =  year - yearOfBirth
6}

If there is a class with the same name as our objects the class is named companion class and the object is named companion object. class or object can access the private members of its companion. Take a look at the code below:

 1// Companion class
 2class Person(val firstName: String, val lastName: String, val yearOfbirth: Int, val address: String) {
 3   import Person.calculateAge
 4  val fullName = firstName + " " + lastName
 5  def this(firstName: String, lastName: String, year: Int) = this(firstName, lastName, year ,"")
 6  def printFullName = println(fullName)
 7  def age: Int = calculateAge(yearOfbirth)
 8}
 9
10// Companion object
11object Person {
12  import java.time.LocalDate
13  val year = LocalDate.now.getYear
14  private def calculateAge(yearOfBirth:Int): Int =  year - yearOfBirth
15}

Objects can have constructors(apply) which take argument and create objects or extractors(unapply), which extracts object values(arguments) and give them back.

 1class PersonObj(firstName: String, lastName: String, yearOfbirth: Int, address: String){
 2  override def toString = s"Person $firstName $lastName born $yearOfbirth and live on $address"
 3}
 4// defined class PersonObj
 5
 6object PersonObj {
 7 def apply (firstName: String, lastName: String, yearOfbirth: Int, address: String) =
 8   new PersonObj(firstName: String, lastName: String, yearOfbirth: Int, address: String)
 9
10  def unapply (personObj: PersonObj): Option[(PersonObj)] = Some(personObj)
11}
12// defined object PersonObj
13
14val person = PersonObj("Peter", "Gabriel", 1970, "")
15
16//person: PersonObj = Person Peter Gabriel born 1970 and live on
17
18PersonObj.unapply(person)
19//res5: Option[PersonObj] = Some(Person Peter Gabriel born 1970 and live on )

On code above, the unapply method returns Some(our_object) and our_object will return the default method, toString, which has been overridden.

A better example is the scala List object, so let’s check apply and unapply methods:

In the above code, pay attention to how unapply is invoking UnapplySeqWrapper.apply getting to the indexed value of the collection. As a proper extractor in case of List, its extractor extracts an specific index of a value introduced as an argument.
